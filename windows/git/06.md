<h1>Git - Atualizar o conteúdo de um arquivo</h1>



Quando um arquivo já está versionado, você pode **atualizá-lo** e registrar essas mudanças no histórico do repositório.

1. No **Visual Studio Code**, abra o arquivo **spring.txt** e atualize o seu conteúdo:

- **spring.txt:** Spring é um framework Java.

2. Para verificar as alterações realizadas no arquivo, utilize o comando:

```bash
git diff
```

3. O comando **git diff** mostra as modificações que ainda não foram adicionadas ao índice (*staging*). Ele indica exatamente quais linhas foram alteradas, como mostra a imagem abaixo:

<div align="left"><img src="https://i.imgur.com/bw6Sewy.png" title="source: imgur.com" /></div>

4. Para checar o status do repositório, use:

```bash
git status
```

5. O retorno mostrará que o arquivo **spring.txt** foi modificado, mas ainda **não está no índice**, ou seja, ainda não foi preparado para commit:

<div align="left"><img src="https://i.imgur.com/xETINlK.png" title="source: imgur.com" /></div>

6. Para adicionar **apenas o arquivo spring.txt** ao índice (*staging*), digite:

```bash
git add spring.txt
```

7. Verifique novamente o status:

```bash
git status
```

8. Agora o Git indicará que o arquivo **spring.txt** está pronto para o próximo commit:

<div align="left"><img src="https://i.imgur.com/XC8uxsD.png" title="source: imgur.com" /></div>

9. Para registrar o commit com a alteração, utilize:

```bash
git commit -m "Meu terceiro commit"
```

10. Confirme que não há mais alterações pendentes:

```bash
git status
```

11. O comando **git status** retornará que não existem arquivos aguardando para serem "commitados", como mostra a imagem abaixo:

<div align="left"><img src="https://i.imgur.com/BOIgZ5Y.png" title="source: imgur.com" /></div>

12. Para visualizar o histórico de commits, utilize:

```bash
git log
```

13. O histórico mostrará todos os commits, e o **HEAD** agora aponta para o `"Meu terceiro commit"`:

<div align="left"><img src="https://i.imgur.com/RtYyCxQ.png" title="source: imgur.com" /></div>

## ✅Tabela Comparativa: `git diff`, `git status` e `git log`

Para trabalhar de forma eficiente com o Git, é importante compreender a função de cada um destes 3 comandos no fluxo de versionamento:

| Comando        | Função                                                       | Quando usar                                                  | Exemplo de saída                                             |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **git diff**   | Mostra as **diferenças** entre o Working Directory e o Staging Area, ou entre commits. | Quando você quer ver **o que mudou** nos arquivos antes de adicioná-los ao índice. | Linhas adicionadas (+) ou removidas (-) em arquivos modificados. |
| **git status** | Mostra o **status atual do repositório**: arquivos modificados, adicionados ao staging ou ainda não rastreados. | Para verificar rapidamente **o que está pronto para commit** e o que ainda precisa ser adicionado. | Lista de arquivos modificados, adicionados ou não rastreados. |
| **git log**    | Exibe o **histórico de commits** do repositório, com hash, autor, data e mensagem do commit. | Para consultar **o histórico de alterações** e localizar commits específicos. | Sequência de commits, geralmente com hash abreviado, autor, data e mensagem. |
